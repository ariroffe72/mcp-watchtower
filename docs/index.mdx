MCP Watchtower is a static analysis tool that checks your MCP (Model Context Protocol) server's tool definitions for problems that silently degrade LLM routing accuracy — duplicate names, inconsistent conventions, parameter conflicts, and manipulative description patterns. Run it in seconds against any local or remote MCP server, or drop it into your CI pipeline as a quality gate.

## How it works

<Steps>
  <Step title="Point at your server">
    Pass a startup command, a remote URL, or a tools manifest JSON file.
  </Step>
  <Step title="Watchtower fetches the tool list">
    It connects to your MCP server, retrieves the full tool manifest, and disconnects — no traffic is intercepted or modified.
  </Step>
  <Step title="All checks run offline">
    Five deterministic checks run against the tool definitions. No LLM calls, no network requests beyond the initial tool fetch.
  </Step>
  <Step title="Get a compatibility report">
    Findings are printed to the console (or emitted as JSON). Exit code `1` on any critical finding, making it a clean CI gate.
  </Step>
</Steps>
