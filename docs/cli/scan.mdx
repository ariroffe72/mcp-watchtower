The `scan` command is the single entry point for running MCP Watchtower. You point it at an MCP server using one of three explicit input modes — `--server`, `--remote`, or `--manifest` — or pipe a tools JSON document over stdin. Watchtower fetches the tool list, runs all five static checks, and prints a human-readable report (or JSON) to stdout. Exit code `1` signals at least one critical finding, making it a clean gate for CI pipelines.

## Synopsis

```bash
npx mcp-watchtower scan [options]
```

## Input modes

You must provide exactly one input source per invocation. Passing more than one of `--server`, `--remote`, or `--manifest` together is an error.

<Warning>
  Providing more than one of `--server`, `--remote`, or `--manifest` in the same command causes an immediate error: `Provide only one input source: --server, --remote, or --manifest.`
</Warning>

## Mode details

### `--server` mode

Use `--server` (or `-s`) to pass the shell command that starts your MCP server. Watchtower spawns the process over stdio, calls `listTools`, collects the results, and then shuts the process down.

```bash
npx mcp-watchtower scan --server "python my_server.py"
npx mcp-watchtower scan --server "node dist/server.js"
npx mcp-watchtower scan --server "uvx my-published-server"
```

The server name in the report is derived automatically from the last segment of the command (e.g. `my_server.py` → `my_server`). Override it with `--name`.

### `--remote` mode

Use `--remote` (or `-r`) to connect to a remote MCP server over HTTP. You must also supply `--auth-token` with a valid bearer token; omitting it throws an error before any network request is made.

<Warning>
  `--auth-token` is required whenever you use `--remote`. Omitting it causes: `Missing auth token. Use --auth-token when connecting with --remote.`
</Warning>

```bash
npx mcp-watchtower scan \
  --remote "https://api.example.com/mcp" \
  --auth-token "$MCP_TOKEN"
```

The server name defaults to the hostname (e.g. `api.example.com`). If the URL path contains a meaningful segment other than `mcp`, that segment is appended: `api.example.com-v2`.

### `--manifest` mode

Use `--manifest` (or `-m`) to read a local JSON file that already contains the tool definitions. This is the recommended mode for CI environments where spinning up a live server is not practical.

```bash
npx mcp-watchtower scan --manifest ./tools.json
```

The JSON must be a raw array of tools, an object with a `tools` array, or an object with a `result.tools` array:

```json
[
  { "name": "get_ticker", "description": "Fetch a stock quote.", "inputSchema": {} }
]
```

### stdin mode

If you do not pass `--server`, `--remote`, or `--manifest`, and stdin is not a TTY, Watchtower reads the tools JSON from stdin. This lets you pipe output from another tool directly into a scan.

```bash
cat tools.json | npx mcp-watchtower scan
curl -s https://api.example.com/tools.json | npx mcp-watchtower scan --name remote-export
```

<Note>
  stdin mode is triggered automatically when stdin is not a TTY. If you run `npx mcp-watchtower scan` interactively with no flags and no pipe, you will see an error listing the available input modes.
</Note>

## Full usage

```
Usage: mcp-watchtower scan [options]

Options:
  -s, --server <command>     command to start the MCP server process
  -r, --remote <url>         remote MCP endpoint URL
  -t, --auth-token <token>   bearer token for --remote MCP endpoint
  -m, --manifest <path>      path to a JSON file containing tools (CI fallback)
  -n, --name <name>          server name for the report
  -j, --json                 output results as JSON
  -p, --platform             platform mode: elevates name collision severity to critical
      --max-tools <number>   maximum tools before warning (default: 20)
  -h, --help                 display help for command
```

## Examples

<CodeGroup>

```bash Local server (Python)
npx mcp-watchtower scan --server "python my_server.py"
```

```bash Local server (Node.js)
npx mcp-watchtower scan --server "node dist/server.js"
```

```bash Local server (uvx)
npx mcp-watchtower scan --server "uvx my-published-server"
```

```bash Remote endpoint
npx mcp-watchtower scan \
  --remote "https://api.example.com/mcp" \
  --auth-token "$MCP_TOKEN"
```

```bash Remote with custom name
npx mcp-watchtower scan \
  --remote "https://api.example.com/mcp" \
  --auth-token "$MCP_TOKEN" \
  --name prod-api
```

```bash Manifest file
npx mcp-watchtower scan --manifest ./tools.json
```

```bash JSON output
npx mcp-watchtower scan --server "node dist/server.js" --json
```

```bash Platform mode
npx mcp-watchtower scan --server "node dist/server.js" --platform
```

```bash Custom tool threshold
npx mcp-watchtower scan --server "node dist/server.js" --max-tools 15
```

```bash Stdin pipe
cat exported-tools.json | npx mcp-watchtower scan --name my-server
```

</CodeGroup>
