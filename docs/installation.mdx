---
title: "Install MCP Watchtower on your machine or project"
description: "Install MCP Watchtower globally with npm, add it as a project dev dependency, or run it on demand with npx — requires Node.js 18 or later."
---

MCP Watchtower is an npm package that runs in Node.js. You can install it globally for command-line use, add it as a dev dependency to a project, or run it directly with `npx` without installing anything at all. All three approaches require Node.js 18 or later.

## Check your Node.js version

Before installing, confirm you're running Node.js 18 or later:

```bash
node --version
```

If the output is below `v18.0.0`, update Node.js before continuing. You can download the latest version from [nodejs.org](https://nodejs.org) or use a version manager like `nvm` or `fnm`.

## Installation options

<CodeGroup>

```bash Global install (npm)
npm install -g mcp-watchtower
```

```bash Dev dependency
npm install --save-dev mcp-watchtower
```

```bash Run without installing (npx)
npx mcp-watchtower scan --server "node dist/server.js"
```

</CodeGroup>

### Global install

Installing globally makes the `mcp-watchtower` command available everywhere on your machine. This is the best choice if you want to scan servers from any directory without a project setup.

```bash
npm install -g mcp-watchtower
mcp-watchtower scan --server "python my_server.py"
```

### Dev dependency

Adding MCP Watchtower as a dev dependency pins a specific version to your project and lets you run it through npm scripts. This is recommended for teams who want consistent versions across environments.

```bash
npm install --save-dev mcp-watchtower
```

Then add a script to your `package.json`:

```json
{
  "scripts": {
    "lint:mcp": "mcp-watchtower scan --server \"node dist/server.js\""
  }
}
```

Run it with:

```bash
npm run lint:mcp
```

### npx (no install)

`npx` downloads and runs the latest version of MCP Watchtower without installing it permanently. This is the quickest way to try it out, and it works well in CI pipelines where you always want the latest version.

```bash
npx mcp-watchtower scan --server "node dist/server.js"
```

<Note>
  `npx` caches the package after the first run, so subsequent runs in the same environment are fast.
</Note>

## Verify the installation

After installing globally or as a dev dependency, confirm Watchtower is available:

```bash
mcp-watchtower --version
```

You should see the version number printed to your terminal (`0.1.0` or later).

## CI usage

In a CI pipeline, `npx` is usually the simplest approach. Here's a GitHub Actions example:

```yaml
- name: Lint MCP tools
  run: npx mcp-watchtower scan --server "node dist/server.js" --json
```

<Tip>
  Use `--json` in CI so that scan results are machine-readable and can be captured by your pipeline's reporting tools.
</Tip>

<Warning>
  MCP Watchtower requires Node.js 18 or later. If your CI runner uses an older version, the `npx` invocation will fail with a Node.js version error.
</Warning>
